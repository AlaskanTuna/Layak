04 / 15 · What Layak Does
Product in one sentence

Endless online searches, replaced with one upload page.

Layak is a grounded agentic concierge: it reads a user's documents, matches them against Budget-2026-gazetted rules, and prepares pre-filled draft packets for all eligible schemes — each one cited to the exact source PDF.

It is explicitly not a chatbot. There is no free text answer. Every number comes from a typed rule engine backed by retrieval, and every packet has a DRAFT — NOT SUBMITTED disclaimer, to ensure manual user verification before submission.

IN →
IC + Payslip + Utility bill or · Manual entry form
⏱
~30–60 seconds, 5 visible agent steps streamed via SSE to the UI
OUT →
Ranked schemes · Annual RM upside · 5 draft PDFs BK-01 · JKM Warga Emas & BKK · LHDN relief summary · STR 2026

07 / 15 · Data Privacy by Design
PDPA 2010-aligned · zero-retention by default

Privacy is the primary design constraint.

01 · DATA MINIMISATION
Only eligibility-relevant fields.
Layak reads the fields each rule actually needs — income, dependants, household size — and nothing more. Uploaded documents are parsed in-memory for one request, then discarded. No PII database, no analytics pipeline, no shadow profile.
NFR-3 · §7 TRD
02 · STATELESS PIPELINE
Documents never leave the request.
Uploaded IC, payslip, and utility-bill files live in memory for the HTTP request only — and are discarded the moment extraction returns.
v1 stateless · v2 unchanged
03 · USER RIGHTS
Export + delete, end-to-end.
Explicit PDPA consent at sign-up, a JSON export endpoint, and a hard-delete endpoint that cascades every evaluation.
FR-17 · FR-12 · FR-18
04 · 30-DAY RETENTION
Free-tier history auto-prunes.
Cloud Scheduler fires a Cloud Run Job every night at 02:00 MYT that deletes free-tier evaluations older than thirty days.
NFR-8 · Nightly prune
05 · NO LIVE SUBMISSION
Draft only. You submit.
Layak never submits on the citizen's behalf. Every output is watermarked DRAFT — NOT SUBMITTED on every page.
FR-8 · Disclaimer §7 PRD
06 · OWNER-GATED READS
Firestore rules enforce it.
All writes are backend-only; client reads require request.auth.uid == resource.data.userId. A user can only see their own evaluations.
TRD §7 · Security rules

13 / 15 · Safety & Compliance
What a grounded public-sector agent must follow

The rules we built for the agent.

01
No live submission — ever.
Layak never writes to any government portal. Outputs are drafts; the citizen submits through the stated official channel.
02
No unverified claim appears in the UI.
If Vertex AI Search returns no passage, the rule is flagged unverified and does not contribute to the ranked list.
03
Synthetic demo documents only.
The MyKad, payslip, and utility-bill specimens are fully fictional and watermarked SYNTHETIC — FOR DEMO ONLY.
04
No final legal determination is claimed.
Every explanation says "you appear to qualify … the agency confirms on application."
05
AI disclosure is explicit.
README names Claude Code (Anthropic) as the primary agentic coding assistant, per hackathon Rules §4.2, which is what allowed us to build and ship Layak within a week.
