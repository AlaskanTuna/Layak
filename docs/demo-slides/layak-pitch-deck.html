03 / 16 · What Layak Does
Product in one sentence

Endless online searches, replaced with one upload page.

A grounded agentic concierge — reads your documents, matches them against Budget-2026 rules, and drafts pre-filled application packets cited to the source PDF.

Not a chatbot. No free-text answers. Every number comes from a typed rule engine, and every packet is watermarked DRAFT — NOT SUBMITTED.

IC + Payslip + Utility bill or · Manual entry form
⏱
~30–60 seconds, 6 visible agent steps streamed via SSE to the UI
OUT →
Ranked schemes · Annual RM upside · 3 draft PDFs BK-01 (STR 2026) · JKM18 (Warga Emas) · LHDN Form B relief summary

12 / 16 · Data Privacy by Design
PDPA 2010-aligned · zero-retention by default

Privacy is the primary design constraint.

01 · DATA MINIMISATION
Only eligibility-relevant fields.
  • Reads only income, dependants, household size.
  • No PII database, no analytics pipeline.
02 · STATELESS UPLOADS
Raw files never touch storage.
  • MyKad, payslip, bill extracted in-memory once.
  • Bytes dropped the moment extraction returns.
03 · USER RIGHTS
Export + delete, end-to-end.
  • Explicit PDPA consent on sign-up.
  • JSON export + hard-delete cascades.
04 · 30-DAY RETENTION
Free-tier history auto-prunes.
  • Nightly Cloud Run Job at 02:00 MYT.
  • Evaluations deleted in 30 days.
05 · NO LIVE SUBMISSION
Draft only. You submit.
  • Never writes to a government portal.
  • Every page watermarked DRAFT — NOT SUBMITTED.
06 · OWNER-GATED READS
You only see your own data.
  • Every doc carries its owner's uid.
  • Firestore rules deny reads when auth.uid ≠ userId.
